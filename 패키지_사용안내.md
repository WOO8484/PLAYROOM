# 패키지 사용 안내

이 ZIP은 GitHub Actions 빌드가 성공한 현재 전체 프로젝트를 기준으로 구성했습니다.

## 클로드에 전달

ZIP 전체를 업로드한 뒤 아래처럼 지시합니다.

```text
압축파일 루트의 `CLAUDE_작업지시서_v1.0.md`를 처음부터 끝까지 읽고 그대로 실행하세요.
현재 프로젝트를 새로 만들지 말고 기존 파일을 직접 수정하세요.
빌드 성공과 APK 생성 확인 전에는 완료로 보고하지 마세요.
최종 결과는 지시서에 적힌 루트 구조의 ZIP 하나로 전달하세요.
```

## GitHub 업로드

클로드 작업 완료 ZIP을 GitHub 저장소 `WOO8484/PLAYROOM`의 루트에 덮어씁니다.

업로드 후 루트에서 다음 항목이 바로 보여야 합니다.

- `.github`
- `app`
- `core`
- `feature`
- `gradle`
- `gradlew`
- `settings.gradle.kts`

프로젝트 폴더가 한 겹 더 들어가면 GitHub Actions가 루트를 찾지 못하므로 금지합니다.

## Actions 확인

`Build Android APK With Full Logs` 실행이 성공하면 Artifacts에서 다음을 확인합니다.

- `StarlightExpedition-debug-apk`
- `StarlightExpedition-full-build-logs`
