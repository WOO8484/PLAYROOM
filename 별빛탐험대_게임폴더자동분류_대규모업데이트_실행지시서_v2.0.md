# 별빛 탐험대 Android 게임 폴더 자동 분류·게임 라이브러리 통합 실행지시서 v2.0

## 0. 작업 성격

- 작업 강도: **높음**
- 작업 유형: **대규모 기능 통합**
- 기준 프로젝트: 사용자가 함께 전달하는 최신 `StarlightExpedition_UI_Stabilization_v1.0` 전체 프로젝트
- 목표: 사용자가 상단 우측의 폴더 아이콘으로 게임 폴더를 지정하면, 앱이 하위 파일을 검색하고 플랫폼을 판별하여 실제 게임 목록으로 저장한 뒤 빠른 시작·홈·즐겨찾기·게임리스트에 같은 데이터를 연결한다.
- 이번 작업은 GUI를 새로 디자인하는 작업이 아니다. 기능 연결에 필요한 최소 UI만 추가하고 현재 확정 GUI를 유지한다.

# 1. 가장 중요한 작업 원칙

1. 새 Android 프로젝트를 만들지 않는다.
2. 전달받은 기존 프로젝트를 직접 수정한다.
3. WebView로 변경하지 않는다.
4. 기존 멀티모듈 구조를 유지한다.
5. 기존 AGP, Kotlin, Compose BOM, Gradle Wrapper 버전을 임의로 올리지 않는다.
6. 기능을 삭제하거나 주석 처리해서 빌드만 통과시키지 않는다.
7. 샘플 게임 데이터가 실제 기능처럼 계속 표시되게 두지 않는다.
8. 실제 컴파일을 하지 못한 경우 컴파일 성공이라고 보고하지 않는다.
9. APK 생성은 Claude 환경의 필수 완료 조건이 아니다. 최종 APK 검증은 GitHub Actions에서 수행한다.
10. 최종 결과물에는 GitHub Actions, Gradle Wrapper, 전체 모듈, 리소스가 빠짐없이 포함되어야 한다.
11. 페이지 전환 잔상 제거, 현재 하단 안전 여백, 현재 색상 안정화 결과를 되돌리지 않는다.
12. 이번 작업과 관계없는 카드 크기, 버튼 위치, 상단 로고, 하단 메뉴 구조를 변경하지 않는다.

# 7. Android 폴더 선택 방식

## 7.1 필수 방식

Compose에서 다음 계약을 사용한다.

```kotlin
ActivityResultContracts.OpenDocumentTree()
```

사용자가 선택한 결과는 절대 일반 파일 경로 문자열로 변환하지 않는다.

잘못된 방식:

```text
/storage/emulated/0/ROMs/...
File(uri.path)
실제 경로 강제 변환
MANAGE_EXTERNAL_STORAGE
```

올바른 기준:

```text
content:// URI
ContentResolver
DocumentsContract
persistable URI permission
```

## 7.2 권한 유지

폴더 선택 성공 직후 읽기 권한을 유지한다.

```kotlin
contentResolver.takePersistableUriPermission(
    uri,
    Intent.FLAG_GRANT_READ_URI_PERMISSION
)
```

쓰기 권한은 이번 작업에 필요하지 않으므로 요청하지 않는다.

Manifest에 아래 권한을 추가하지 않는다.

```text
MANAGE_EXTERNAL_STORAGE
WRITE_EXTERNAL_STORAGE
```

SAF만 사용할 경우 광범위 저장소 권한을 요구하지 않는다.

## 7.3 권한 확인

앱 시작 및 검색 시작 전에 다음을 확인한다.

```kotlin
contentResolver.persistedUriPermissions
```

등록된 폴더 URI에 읽기 권한이 없으면:

- 게임 목록을 즉시 삭제하지 않는다.
- 해당 폴더 상태를 `PERMISSION_LOST`로 표시한다.
- “폴더 권한을 다시 연결해 주세요” 안내를 보여준다.
- 사용자가 같은 폴더를 다시 선택할 수 있게 한다.

## 7.4 Android 제한 안내

폴더 선택기가 다음 위치 선택을 허용하지 않는 상황을 오류로 오해하지 않는다.

- 저장소 최상위 루트
- 다운로드 폴더 자체
- `Android/data`
- `Android/obb`

사용자에게는 ROM이 들어 있는 별도 하위 폴더를 선택하도록 안내한다.

# 8. 상단 우측 폴더 아이콘

## 8.1 위치

기존 상단 헤더 우측에 게임 폴더 관리 아이콘을 추가한다.

```text
[폴더 아이콘] [설정 아이콘]
```

- 설정 아이콘이 표시되는 화면: 폴더 아이콘을 설정 아이콘 왼쪽에 둔다.
- 빠른 시작처럼 설정 아이콘이 숨겨진 화면: 폴더 아이콘만 우측에 표시한다.
- 폴더 아이콘은 모든 주요 탭에서 접근 가능해야 한다.
- 기존 상단 로고, 제목, 헤더 높이는 변경하지 않는다.
- 아이콘 버튼 크기는 기존 설정 버튼과 동일하게 맞춘다.
- 접근성 설명은 `게임 폴더 관리`로 지정한다.

권장 아이콘:

```text
FolderOpen
Folder
CreateNewFolder
```

화면 안에 텍스트 버튼을 새로 추가하지 않는다.

## 8.2 클릭 동작

등록 폴더가 없는 경우:

1. 폴더 관리 다이얼로그를 연다.
2. “게임 폴더 추가”를 누르면 시스템 폴더 선택기를 연다.

등록 폴더가 있는 경우:

- 등록 폴더 목록
- 폴더 추가
- 전체 다시 검색
- 개별 다시 검색
- 폴더 연결 해제
- 마지막 검색 시간
- 검색된 게임 수
- 권한 상태

를 중앙 고정 다이얼로그에서 보여준다.

바텀시트로 만들지 않는다.

# 9. 폴더 데이터 모델

`core:model`에 Android 타입을 직접 노출하지 않는다. URI는 문자열로 저장한다.

```kotlin
data class GameSourceFolder(
    val id: String,
    val treeUri: String,
    val displayName: String,
    val addedAtEpochMillis: Long,
    val lastScannedAtEpochMillis: Long?,
    val lastKnownGameCount: Int,
    val enabled: Boolean,
    val permissionState: FolderPermissionState,
    val lastErrorMessage: String?
)
```

권장 상태:

```kotlin
enum class FolderPermissionState {
    GRANTED,
    LOST,
    UNKNOWN
}
```

폴더 ID는 URI 문자열을 그대로 노출하지 말고 안정적인 해시로 만든다.

```text
SHA-256(treeUri)
```

이 해시는 파일 내용을 읽는 해시가 아니다.

    val documentUri: String,

# 12. 플랫폼 판별 결과

플랫폼만 반환하지 말고 신뢰도와 이유를 함께 반환한다.

```kotlin
data class PlatformClassification(
    val platform: Platform,
    val confidence: ClassificationConfidence,
    val reason: String,
    val launchable: Boolean
)

enum class ClassificationConfidence {
    HIGH,
    MEDIUM,
    LOW,
    UNKNOWN
}
```

판별 근거 예:

```text
확장자 .gba
상위 폴더명 PSP
동일 이름의 .cue 존재
MAME 폴더 안의 .zip
모호한 .iso이고 폴더 힌트 없음
```

신뢰도가 낮으면 억지로 플랫폼을 확정하지 말고 `UNKNOWN`으로 둔다.

# 13. 플랫폼 판별 규칙

## 13.1 확장자만으로 높은 신뢰도 판별 가능

| 플랫폼 | 확장자 |
|---|---|
| NES | `.nes` |
| SNES | `.sfc`, `.smc` |
| Game Boy | `.gb` |
| Game Boy Color | `.gbc` |
| Game Boy Advance | `.gba` |

## 13.2 보조 정보가 필요한 모호한 확장자

### `.iso`

다음 폴더명 힌트를 함께 확인한다.

```text
psp
playstation portable
ps2
playstation 2
gamecube
game cube
gc
wii
```

폴더 힌트가 없으면 `UNKNOWN`으로 둔다.  
파일명만 보고 PSP 또는 PS2로 단정하지 않는다.

### `.chd`

다음 폴더명 힌트와 함께 판별한다.

```text
ps1 / psx
ps2
dreamcast / dc
saturn
segacd / mega cd
pcengine / turbografx
mame / arcade
```

힌트가 없으면 `UNKNOWN`으로 둔다.

### `.pbp`

```text

PS1/PSX/EBOOT 힌트 → PlayStation 1
힌트 없음 → UNKNOWN
```

### `.bin`

`.bin` 단독 파일을 무조건 게임으로 등록하지 않는다.

우선순위:

1. 동일 기본 이름의 `.cue`가 있으면 `.bin`은 동반 파일로만 처리
2. Genesis 폴더 힌트가 있으면 Genesis 후보
3. BIOS 폴더 또는 BIOS 이름 패턴이면 제외
4. 그 외에는 `UNKNOWN` 또는 검색 제외

### `.cue`

부모 폴더 힌트로 다음을 구분한다.

```text
PS1 / PSX
Sega Saturn
Sega CD / Mega CD
PC Engine CD
```

힌트가 없으면 확정하지 않고 `UNKNOWN`으로 둔다.

### `.zip`, `.7z`

압축을 풀거나 내부 파일을 읽지 않는다.

다음 폴더 힌트가 명확한 경우에만 Arcade로 분류한다.

```text
mame
fbneo
finalburn
arcade
```

그 외 ZIP/7Z는 게임으로 자동 등록하지 않거나 `UNKNOWN_ARCHIVE` 성격으로 제외 목록에 기록한다.

### `.cia`

Nintendo 3DS로 분류하되 설치 패키지이므로:

```text
platform = NINTENDO_3DS
launchable = false
reason = "Nintendo 3DS 설치 패키지"
```

# 16. DocumentsContract 재귀 검색

일반 파일 API를 사용하지 않는다.

권장 흐름:

1. `DocumentsContract.getTreeDocumentId(treeUri)`
2. `DocumentsContract.buildChildDocumentsUriUsingTree(...)`
3. `ContentResolver.query(...)`
4. `Cursor.use { ... }`
5. 디렉터리면 재귀 또는 반복 큐에 추가
6. 파일이면 메타데이터 후보로 변환

조회 권장 열:

```text
Document.COLUMN_DOCUMENT_ID
Document.COLUMN_DISPLAY_NAME
Document.COLUMN_MIME_TYPE
Document.COLUMN_SIZE
Document.COLUMN_LAST_MODIFIED
Document.COLUMN_FLAGS
```

주의:

- 크기와 수정 시간은 null일 수 있다.
- provider가 일부 열을 제공하지 않아도 실패하지 않는다.
- 모든 Cursor를 `use`로 닫는다.
- 파일 전체 내용을 메모리에 올리지 않는다.
- UI 스레드에서 재귀 검색하지 않는다.
- 동일 documentId 무한 반복을 막기 위해 방문 ID 집합을 둔다.
- 코루틴 취소 시 `ensureActive()`로 빠르게 중단한다.
- 20~50개 항목마다 진행 상태를 갱신한다.
- 진행 상태 갱신을 파일 하나마다 발생시켜 UI를 과도하게 재구성하지 않는다.

# 17. 검색 실행 생명주기

이번 버전은 새 WorkManager 의존성을 추가하지 않는다.

대신 Application 수명 범위의 검색 코디네이터를 둔다.

권장:

```kotlin
CoroutineScope(SupervisorJob() + Dispatchers.IO)
```

이 Scope는 `AppContainer` 또는 전용 `GameScanCoordinator`가 소유한다.

요구사항:

- 다이얼로그를 닫아도 검색 계속
- 탭을 이동해도 검색 계속
- 화면 회전에도 ViewModel 상태 복원
- 동시에 두 검색 실행 금지
- 다시 검색 요청 시 현재 작업 취소 여부 확인
- 앱 프로세스 종료 시 검색이 중단될 수 있음
- 중단 시 기존 게임 목록은 유지
- 다음 실행에서 상태를 `INTERRUPTED`로 안내하고 다시 검색 가능

주기적 자동 검색은 구현하지 않는다.

# 20. 묶음 파일 처리

## 20.1 CUE/BIN

```text
game.cue
game.bin
```

위 구조에서는 게임 1개만 등록한다.

- 대표 URI: `.cue`
- `.bin`: companionUris
- `.bin` 별도 게임 등록 금지

CUE 내용을 읽는 경우:

- 전체 파일을 읽지 않음
- 작은 텍스트 범위만 읽음
- `FILE` 항목의 동반 파일명 확인
- 읽기 실패 시 동일 기본 이름 규칙으로 폴백

## 20.2 GDI

```text
game.gdi
track01.bin
track02.raw
```

- 대표 URI: `.gdi`
- 트랙 파일 별도 등록 금지

## 20.3 M3U

M3U가 존재하면 다중 디스크 대표 파일로 우선한다.

```text
game.m3u
game (Disc 1).cue
game (Disc 2).cue
```

- 대표 게임: `.m3u`
- 포함된 CUE는 별도 목록에서 중복 노출하지 않음
- M3U 파싱 실패 시 기존 CUE 검색으로 폴백
- 파일 경로가 폴더 밖을 가리키면 안전하게 무시

## 20.4 다중 디스크

이번 버전에서는 복잡한 디스크 전환 UI를 만들지 않는다.

저장 가능한 정보:

```text
discIndex
discCount
groupKey
companionUris
```

게임리스트에는 대표 항목 1개를 우선 표시한다.

# 21. 중복 제거와 안정적 ID

## 21.1 게임 ID

전체 ROM 해시를 계산하지 않는다.

권장 ID:

```text
SHA-256(sourceFolderId + "|" + documentId)
```

documentId를 얻지 못하는 provider에서는:

```text
SHA-256(sourceFolderId + "|" + documentUri)
```

## 21.2 재검색

같은 폴더를 다시 검색할 때:

- 같은 ID → 기존 항목 업데이트
- 새 ID → 신규 추가
- 이전에 있었지만 이번 성공 검색에 없는 ID → 해당 폴더에서 삭제
- 즐겨찾기와 플레이 기록 → 같은 ID면 유지

중요:

검색이 실패하거나 취소되면 이전 목록을 삭제하지 않는다.

## 21.3 여러 폴더 중복

동일 URI가 여러 루트 검색에 중복 등장하면 하나만 유지한다.

제목만 같다고 자동 삭제하지 않는다.

다른 파일이지만 제목·플랫폼·크기가 같은 경우:

- 자동 삭제 금지
- 잠재 중복 표시용 플래그만 둘 수 있음
- 이번 UI에서 별도 중복 관리 화면은 만들지 않음

# 22. 로컬 저장 방식

Room과 KSP를 추가하지 않는다.

## 22.1 폴더 설정

등록 폴더 URI와 기본 설정은 기존 DataStore를 확장해 저장한다.

여러 폴더는 JSON 문자열 또는 명확한 직렬화 형식으로 저장한다.

## 22.2 게임 라이브러리

게임 수가 많을 수 있으므로 각 게임을 DataStore key로 나누지 않는다.

앱 내부 저장소에 버전이 있는 JSON 파일로 저장한다.

권장 파일:

```text
filesDir/starlight/game_library_v1.json
```

권장 최상위 구조:

```json
{
  "schemaVersion": 1,
  "savedAtEpochMillis": 0,
  "games": [],
  "folders": []
}
```

## 22.3 안전 저장

- 임시 파일에 먼저 저장
- 쓰기 완료 후 원본 교체
- 쓰기 도중 실패하면 기존 파일 유지
- `Mutex`로 동시 쓰기 차단
- JSON 파싱 실패 시 파일을 즉시 덮어쓰지 않음
- 손상 파일을 별도 백업 이름으로 이동
- 오류 상태를 UI에 표시

DTO와 화면 모델을 분리한다.

`Instant`는 DTO에서 epoch millis로 저장한다.

# 23. 로컬 커버 자동 매칭

이 영역은 장식 이미지가 아니라 실제 게임 커버 위치다.

## 23.1 자동 교대 제거

현재 `QuickStartScreen`의 다음 동작을 제거한다.

```text
3.5초 타이머
coverIndex
LaunchedEffect 반복
두 리소스 Crossfade 자동 교대
게임과 무관한 임의 커버 전환
```

게임이 변경될 때만 해당 게임의 커버가 변경되어야 한다.

추천 게임을 랜덤 선택하면 추천 게임 커버로 바뀐다.  
시간이 지났다는 이유로 커버가 바뀌면 안 된다.

## 23.2 커버 검색 위치

다음 위치를 확인한다.

1. ROM과 같은 폴더
2. 선택한 루트의 `covers`
3. 선택한 루트의 `cover`
4. 선택한 루트의 `boxart`
5. 선택한 루트의 `artwork`
6. 선택한 루트의 `images`

## 23.3 지원 이미지

```text
.png
.jpg
.jpeg
.webp
```

## 23.4 매칭 우선순위

1. 정규화된 게임명과 완전 일치
2. ROM 기본 파일명과 완전 일치
3. `-front`, `_front`, `(front)` 제거 후 일치
4. 지역·버전 태그 제거 후 일치
5. 플랫폼 하위 커버 폴더에서 일치

무관한 비슷한 이름을 억지로 연결하지 않는다.

## 23.5 커버 저장

이미지를 앱 내부로 복사하지 않는다.

```text
coverUri = content://...
```

문자열로 저장하고 동일 persistable tree 권한으로 읽는다.

## 23.6 표시

Coil이나 Glide를 추가하지 않는다.

`LocalCoverImageLoader`를 구현해 다음을 지킨다.

- ContentResolver 사용
- IO 스레드에서 디코딩
- 목록 썸네일 크기에 맞게 샘플링
- 전체 원본 이미지를 메인 스레드에서 디코딩 금지
- 작은 LRU 메모리 캐시
- 실패 시 플랫폼 기본 커버
- 화면에서 사라지면 불필요한 로드 취소

API 29 이상에서는 `ContentResolver.loadThumbnail()`을 사용할 수 있다.  
API 26~28은 `openInputStream()`과 샘플링 디코딩으로 폴백한다.

# 25. 실제 게임 저장소 연결

`GameRepositoryImpl`에서 다음 의존을 제거한다.

```text
SampleGameData.games
고정 baseGames
샘플 CONTINUE_GAME_ID
```

생산 코드의 게임 목록은 로컬 게임 라이브러리에서 가져온다.

권장 Repository 인터페이스:

```kotlin
interface GameRepository {
    fun observeGames(): Flow<List<Game>>
    fun observeRecentGames(limit: Int = 3): Flow<List<Game>>
    fun observeGame(gameId: String): Flow<Game?>
    suspend fun setFavorite(gameId: String, favorite: Boolean)
}
```

폴더와 검색은 분리한다.

```kotlin
interface GameFolderRepository {
    fun observeFolders(): Flow<List<GameSourceFolder>>
    fun observeScanState(): StateFlow<GameScanState>

    suspend fun addFolder(treeUri: String, displayName: String): Result<GameSourceFolder>
    suspend fun removeFolder(folderId: String): Result<Unit>

    fun startScan(folderId: String? = null)
    fun cancelScan()
}
```

폴더 삭제 시 확인:

```text
폴더 연결만 해제
해당 폴더에서 등록된 게임 목록 제거
즐겨찾기와 기록도 함께 제거될 수 있음
```

사용자가 취소할 수 있어야 한다.

# 26. RecommendationRepository 수정

현재 추천 저장소가 `SampleGameData`를 직접 사용하면 안 된다.

권장 방식:

```kotlin
interface RecommendationRepository {
    suspend fun recommendNext(
        games: List<Game>,
        excludedGameIds: Set<String> = emptySet()
    ): Recommendation?
}
```

후보가 없으면 null을 반환한다.

추천 조건 1차:

- 실제 등록 게임만 사용
- 미분류 게임도 후보가 될 수 있음
- `isLaunchable == false`인 설치 패키지는 후보에서 제외 가능
- 현재 이어 하기 게임 제외
- 직전 추천 게임 제외
- 후보가 1개면 그 게임 사용

# 27. 빠른 시작 화면 연결

## 27.1 이어 하기 카드

- `lastPlayedAt`이 가장 최근인 실제 게임 사용
- 실제 최근 게임이 없으면 가짜 “이어 하기” 게임을 만들지 않음
- 최근 게임이 없을 때는 빈 상태 또는 “아직 플레이 기록이 없습니다” 표시
- 커버는 해당 게임의 `coverUri`
- 커버가 없으면 해당 플랫폼 기본 커버

실제 실행 기능이 아직 없으므로 스캔 직후 게임을 “최근 플레이”로 기록하지 않는다.

## 27.2 오늘 뭐 하지 카드

- 실제 등록 게임 중 추천
- 랜덤 선택 시 게임과 커버가 함께 변경
- 게임명, 플랫폼, 커버가 동일 게임을 가리켜야 함
- 다른 게임의 커버 재사용 금지
- 타이머 기반 자동 커버 교대 금지

## 27.3 게임이 없는 상태

두 카드 안에 가짜 샘플 게임을 표시하지 않는다.

권장 안내:

```text
등록된 게임이 없습니다.
상단의 폴더 아이콘에서 게임 폴더를 추가해 주세요.
```

필요하면 카드 내부의 작은 “폴더 선택” 동작을 상단 폴더 관리 다이얼로그와 연결할 수 있다.

확정 카드 크기와 기본 배치는 유지한다.

# 28. 게임리스트 화면 연결

## 28.1 실제 데이터

- 샘플 목록 제거
- `GameRepository.observeGames()` 연결
- 즐겨찾기 즉시 반영
- 앱 재실행 후 유지

## 28.2 플랫폼 필터

현재 장르 필터 칩을 실제 플랫폼 필터로 전환한다.

예:

```text
전체
PSP
PS1
PS2
GBA
NDS
3DS
GameCube
Wii
미분류
```

모든 플랫폼을 항상 표시하지 않는다.

- 현재 게임이 있는 플랫폼만 표시
- `전체`은 항상 표시
- 칩 영역은 가로 스크롤 허용
- 화면 전체 가로 스크롤은 금지

`GameListUiState`:

```kotlin
val selectedPlatform: Platform?
val availablePlatforms: List<Platform>
```

기존 `selectedGenre` 기반 필터는 제거하거나 향후용으로 보존하되 현재 UI와 충돌하지 않게 한다.

## 28.3 검색

검색 대상:

- 표시 게임명
- 원본 파일명
- 원제
- 플랫폼 표시명

한글 검색 정규화와 기존 `SearchUtils`를 재사용한다.

## 28.4 목록 행

각 항목에 최소 표시:

- 실제 커버 또는 플랫폼 대체 커버
- 게임명
- 플랫폼 배지
- 원본 파일명 또는 간단 보조 정보
- 즐겨찾기 버튼

파일 URI 전체를 화면에 노출하지 않는다.

## 28.5 정렬

기본 정렬:

```text
최근 플레이가 있는 게임 우선
그다음 게임명 오름차순
```

최근 기록이 없는 초기 상태에서는 게임명 오름차순을 사용한다.

# 30. 홈 화면 연결

실제 데이터로 계산한다.

- 등록 게임 수
- 즐겨찾기 수
- 최근 플레이 3개
- 플레이 기록이 없으면 적절한 빈 상태
- 샘플 최근 실행 시간 제거
- 스캔한 게임을 실행한 것으로 간주하지 않음

실제 실행 연동 전에는 오늘 플레이 시간이 0분일 수 있다.  
가짜 시간을 넣지 않는다.

현재 홈 카드 구조와 하단 안전 여백을 변경하지 않는다.

# 33. 성능 기준

- 메인 스레드 파일 검색 금지
- 메인 스레드 이미지 디코딩 금지
- 파일 전체 바이트 로드 금지
- 초기 검색에서 전체 SHA/CRC 계산 금지
- 압축 해제 금지
- 검색 결과를 한 파일마다 디스크 저장하지 않음
- 폴더 1개 검색 완료 후 원자적으로 저장
- 큰 목록은 LazyColumn 유지
- 커버 이미지 메모리 캐시 크기 제한
- 화면을 떠난 커버 요청 취소
- 검색 진행 Flow 업데이트 간격 제한

권장 테스트 규모:

```text
게임 0개
게임 1개
게임 100개
게임 1,000개
깊이 8단계 폴더
커버 500개
```

1,000개 파일에서 메모리 전체 원본 이미지를 읽는 구조는 금지한다.

# 34. 컴파일 오류 예방 규칙

Compose 위임 import 확인:

```kotlin
import androidx.compose.runtime.getValue
import androidx.compose.runtime.setValue
```

InteractionSource 경로:

```kotlin
import androidx.compose.foundation.interaction.MutableInteractionSource
```

다른 모듈의 nullable 프로퍼티 스마트 캐스트:

```kotlin
val value = objectFromOtherModule.nullableValue
if (value != null) {
    // value 사용
}
```

금지:

```text
nullable 공개 프로퍼티를 조건 검사 후 직접 스마트 캐스트
사용하지 않는 import 방치
동일 이름의 Platform 충돌
android.net.Uri를 core:model에 저장
Context를 ViewModel 상태에 저장
ContentResolver를 Composable에서 직접 장시간 호출
```

`Result`와 sealed state를 혼용해 오류가 사라지지 않게 한다.
