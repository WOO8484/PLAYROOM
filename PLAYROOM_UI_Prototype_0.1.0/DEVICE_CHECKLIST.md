# DEVICE CHECKLIST

이 프로젝트를 만든 샌드박스 환경에는 Android SDK/에뮬레이터가 없어 아래 항목은
**실제 기기·에뮬레이터에서 직접 확인되지 않았습니다.** Android Studio에서 빌드한
뒤 아래 순서대로 확인해주세요. 코드 상 대응 방식은 각 항목에 함께 적어둡니다.

## 화면 크기

- [ ] 360×640 (소형) — 좌우 여백 14dp, 카드 간격 10dp, 축소 제목 크기 적용
      (`PlayroomSpacing.ScreenHorizontalCompact`, `isCompactWidth()`)
- [ ] 360×800
- [ ] 412×915 (표준) — 좌우 여백 16dp, 카드 간격 12dp
- [ ] 600dp 이상 (태블릿) — 콘텐츠 최대 폭 520dp 중앙 정렬
      (`Modifier.playroomContentWidth()`), 카드가 전체 폭으로 늘어나지 않는지 확인

## 글자 배율 (설정 → 접근성 → 글자 크기)

- [ ] 1.0 (기본)
- [ ] 1.15 — 카드 제목/버튼 문구가 잘리지 않고 말줄임(`TextOverflow.Ellipsis`) 또는
      줄바꿈으로 처리되는지 확인
- [ ] 1.3 이상 — 필요한 화면(팝업 등)만 내부 스크롤되는지 확인
      (`PlayroomDialog`는 `verticalScroll` 적용됨)

## 내비게이션 방식

- [ ] 제스처 내비게이션 — 하단 바가 제스처 바와 겹치지 않는지
      (`PlayroomBottomBar`의 `WindowInsets.safeDrawing` 처리)
- [ ] 3버튼 내비게이션 — 동일 항목 확인

## 시스템 영역

- [ ] 상태바와 상단바 콘텐츠가 겹치지 않는지 (`PlayroomTopBar`)
- [ ] 카메라 홀/노치가 있는 기기에서 좌우 잘림이 없는지

## 기능 확인 (작업지시서 §29 완료 조건과 동일)

- [ ] 첫 실행 시 환영 화면이 뜨고, "PLAYROOM 시작하기" 이후 다시 뜨지 않음
      (설정 → "환영 화면 다시 보기"로 재현 가능)
- [ ] 메인 화면 카드 2개(이어서 하기 / 오늘 뭐 하지) 모두 정상 표시
- [ ] 하단 메뉴 4개(메인/게임룸/즐겨찾기/설정) 탭 전환
- [ ] 게임룸: 전체/즐겨찾기/기종 필터, "폴더 관리" 가짜 스캔(1초 후 완료)
- [ ] 즐겨찾기: 별표 토글이 모든 화면에 즉시 반영
- [ ] 검색: 실시간 필터링, 뒤로가기로 닫힘
- [ ] 설정: 토글·화면 방향 순환·스킨 4종 전환(실제 색상 변경 확인)
- [ ] 게임 상세 팝업, 추천 옵션 팝업
- [ ] 가짜 게임 실행 화면 → 일시정지 메뉴 → 게임 종료
- [ ] 뒤로가기 우선순위: 팝업 닫기 → 일시정지 메뉴 → (플레이 화면이면 일시정지
      메뉴 열기) → 검색 닫기 → 메인 아니면 메인으로 → 메인이면 앱 종료
- [ ] 하단 큰 빈 공간 없음, 가로 스크롤 없음
- [ ] 모든 문구 한국어
- [ ] 크래시 없음
